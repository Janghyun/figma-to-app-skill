# figma-to-app — Figma 링크 1개 → 동일 품질 웹앱

Claude Code 스킬. Figma 링크 하나(디자인 파일·노드 URL, 정리 안 된 드래프트 보드·플립북 스펙 문서 포함)를 받아 **화면 열거 → 계약 고정 → 병렬 포팅 → QA 루프** 파이프라인으로 동작하는 웹앱을 만든다.

## 왜 이 스킬인가

여러 화면을 그냥 병렬로 포팅하면 화면마다 브랜드 색이 미묘하게 다르고(#4337ff vs blue-600), 화면마다 자기 사이드바를 또 만든 앱이 나온다. 이 스킬의 핵심 원칙은 **병렬화 전에 계약을 먼저 고정**하는 것이다:

1. 디자인 토큰 (색·폰트·radius — 임의 hex 금지)
2. AppShell (사이드바+헤더 단일 소유 — 화면은 내용물만)
3. 공통 컴포넌트 API (SpecMarker 등)

계약은 대상 레포의 `CLAUDE.md`에 기록되어 이후 투입되는 모든 포팅 에이전트가 자동으로 읽는다. 스킬 자체는 **절차만** 담는다.

## 파이프라인

| 단계 | 산출물 | 병렬성 |
|---|---|---|
| 0. 접근 확인 | Figma MCP 연결 + 페이지 열거 가능 여부 | - |
| 1. 정찰·카탈로그 | 노드ID·화면명·W×H·섹션 표 + 스크린샷 + **사람 게이트**(정본 세대·상태 변형 범위 확정) | 서브에이전트 1 |
| 2. 기반 계약 | 토큰·AppShell·컴포넌트 API + 대표 화면 1개, 레포 CLAUDE.md | 에이전트 1 |
| 3. 병렬 포팅 | 화면별 페이지 파일 (각자 tsc 통과) | 화면 수만큼 |
| 4. 통합 | 라우팅·2depth 네비·CTA wiring, build 통과 | 직렬 |
| 5. QA 루프 | 전 라우트 무결 — 렌더·콘솔 에러 / 클릭 도달성 / 중복 크롬 (≥2바퀴) | 직렬 반복 |

특기 사항:

- **3중 fallback 정찰**: REST 페이지 열거가 고장난 파일(1페이지만 반환)에서도 `use_figma`로 전체 페이지를 확보하는 검증된 우회 경로 포함.
- **드래프트 보드 대응**: 폭이 다른 세대 혼재·동명 상태 변형·흩어진 메모가 있는 정리 안 된 파일을 좌표 클러스터링으로 카탈로그화.
- **사람 게이트**: 어느 세대가 정본인지, 상태 변형(로딩/실패/빈 상태)을 전부 만들지 대표만 만들지 — 수십 화면을 잘못 만들기 전에 사용자에게 한 번에 확정받는다.
- **SpecMarker**: 원본 플립북의 Description 주석을 화면 위 번호 뱃지+팝오버로 이관해 스펙이 코드에 남는다.

## 요구 사항

- Claude Code + **Figma 공식 MCP 연결** (필수 — 0단계에서 자동 확인, 미연결 시 OAuth 유도)
- QA 단계용 Playwright (없으면 브라우저 MCP로 대체, 스킬이 안내)

## 설치

프로젝트 스코프 (해당 레포를 쓰는 팀원 전원에게 자동 적용, 권장):

```bash
mkdir -p <your-repo>/.claude/skills
cp -r figma-to-app <your-repo>/.claude/skills/
```

개인 전역 (모든 프로젝트에서 사용):

```bash
cp -r figma-to-app ~/.claude/skills/
```

## 사용

```
/figma-to-app https://www.figma.com/design/<fileKey>/<파일명>?node-id=<A-B>
```

또는 Figma 링크를 주며 "이거 웹앱으로 만들어줘"라고 하면 스킬이 자동 트리거된다.

## 산출물 구조 (대상 레포)

```
CLAUDE.md               ← 디자인 토큰·AppShell 소유권·컨벤션 (계약)
docs/figma/catalog.md   ← 화면 카탈로그 (노드ID ↔ 페이지 파일 매핑)
src/components/         ← AppShell, SpecMarker, PlaceholderPage
src/pages/              ← 화면별 페이지 (상태 변형은 로컬 토글/?state=)
scripts/qa*.mjs         ← 재실행 가능한 전 라우트 QA 스크립트
```
